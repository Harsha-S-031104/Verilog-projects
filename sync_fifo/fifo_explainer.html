Bhai, listen up! Don't panic. FIFO is just a queue. If you understand how people queue up at a McDonald's counter, you know FIFO. First customer orders, first customer gets the burger. Let's break it down so you can write the logic on the exam paper tomorrow.

🍔 The Fast Food Counter Analogy:
Imagine a counter with 16 slots (depth). The cashier (Write) puts orders in slots, and the chef (Read) takes them out to cook.
- wr_ptr tells the cashier where to put the NEXT order.
- rd_ptr tells the chef which order is NEXT to cook.
- Pointers move in a circle. When they hit slot 15, they wrap back to 0!

Must-Know Exam Questions

Q1. Why do we need ADDR_WIDTH + 1 bits for Pointers?
If depth is 16, we need 4 bits (0 to 15) to address slots. But if we only use 4 bits, when wr_ptr == rd_ptr, is the FIFO empty or completely full? The address bits look identical!

The Trick: Use 5 bits (MSB is the wrap-around bit).
  • If wr_ptr == rd_ptr (all bits match): FIFO is Empty.
  • If wr_ptr[3:0] == rd_ptr[3:0] but wr_ptr[4] != rd_ptr[4] (lower bits match but MSB differs): FIFO is Full (Write pointer completed one more circle than Read pointer).
Q2. Write Pointer and Read Pointer update logic?

On every clock cycle, we only increment a pointer if we perform a valid operation.

// Write pointer increment logic
always @(posedge clk) begin
  if (rst) wr_ptr <= 0;
  else if (wr_en && !full) wr_ptr <= wr_ptr + 1;
end

Same goes for rd_ptr: only increments if read is enabled and FIFO is not empty!
Q3. How do we calculate flags? (The RTL math)
// Empty flag: when pointers are exactly identical
assign empty = (wr_ptr == rd_ptr);

// Full flag: address matches but MSB wrap-around bits are different
assign full = (wr_ptr[ADDR_WIDTH-1:0] == rd_ptr[ADDR_WIDTH-1:0]) && (wr_ptr[ADDR_WIDTH] != rd_ptr[ADDR_WIDTH]);

For almost_full and almost_empty, we can calculate the current count of elements:
count = wr_ptr - rd_ptr

Q4. What are Overflow and Underflow?
  • Overflow: Writing into a full FIFO. The data gets ignored and we raise a sticky flag to warn the system.
  • Underflow: Reading from an empty FIFO. The output is junk/unstable, and we raise a warning flag.
In your testbench, you must test these boundary conditions to prove your design doesn't crash!
